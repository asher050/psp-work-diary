At the start of week 9, we had our first client meeting, which gave us a much clearer understanding of what they wanted from the project. 
During this meeting, we realised that the system needed to be more than just a registration page. There were additional requirements and expectations that meant the project was broader than we had first assumed.
As a team, we took notes throughout the meeting so that we fully understood what was being asked of us. 
This was important because it allowed us to identify the main tasks that still needed to be completed and helped us avoid misunderstandings later on.
Following the meeting, it became clear that we needed to create a more detailed plan for the second and final sprint week. 
As a group, we now need to decide how the remaining work will be divided, what should be prioritised first, and how we will make sure everything is completed on time.
In preparation for Sprint Two, we also discussed how we would test the success of the application. This will include testing the main user journeys, such as login, attendance registration, and admin functions, to make sure they all work correctly.
We also intend to test how the system handles incorrect inputs, check that the pages display properly on different devices, and confirm that the application works reliably with the API. This will help us measure whether the system meets the client’s requirements and is ready for final presentation.
