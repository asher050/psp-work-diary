Week 2 – GitHub Setup and Client Requirements
In week 2, our team completed the Git/GitHub Recipes lab, which helped us understand how to use GitHub effectively when working in a team. 
Through this activity, we practised creating repositories, committing changes, and managing shared work. Evidence of this can be seen in our team repository:
https://github.com/c3076201/team-recipe-book
This exercise helped improve my understanding of version control and how GitHub can be used to track progress and avoid conflicts when multiple team members are working on the same project.
We also continued discussing the client's requirements. My understanding of the system became clearer as we started identifying the main features needed for the application. 
These included allowing apprentices to log in using their existing GTA credentials and enter a registration code to record their attendance. 
At this stage, we also began drafting a MoSCoW analysis to prioritise the system requirements.
To support collaboration, our team set up GitHub repositories to store our work and track development progress. 
GitHub also allows us to maintain our weekly work diary entries and provides a clear record of contributions from each team member.
